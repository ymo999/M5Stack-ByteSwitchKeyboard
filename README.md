# M5Stack ByteSwitchKeyboard

![Built with Claude](https://img.shields.io/badge/Built%20with-Claude-D97757)

M5Stack Basic + M5Stack用Byteスイッチユニット（STM32G031）を使った、簡易鍵盤（1オクターブ）の練習プログラムです。

M5Stack Basic本体の開発に慣れることを目的とした練習シリーズの一環として作成しました。

## 概要

Byteスイッチユニットの8個のトグルスイッチに、ドレミファソラシド（1オクターブ）を割り当て、スイッチをONにしている間、対応する音階が本体スピーカーから鳴り続けます。同時にユニット上のRGB LEDが虹色に点灯し、視覚的にもどのスイッチが鳴っているか分かるようになっています。

さらに、M5Stack Basic本体の物理ボタンで、半音上げ下げ・オクターブ変更ができます。

## 使用ハードウェア

| 製品名 | 型番 | 備考 |
|---|---|---|
| M5Stack Basic | SKU:K001 | 対象デバイス本体 |
| M5Stack用Byteスイッチユニット | STM32G031搭載 | トグルスイッチ×8、RGB LED×9、I2C接続 |

## 接続方法

Byteスイッチユニットを、M5Stack Basic本体の **Port A**（Groveコネクタ）に接続してください。

| 信号線 | GPIO番号 |
|---|---|
| SDA | GPIO21 |
| SCL | GPIO22 |

**Byteスイッチユニット実機のI2Cアドレスは `0x47` です。**
（Web上の一部情報では`0x46`と紹介されている場合がありますが、本プロジェクトの実機検証では`0x47`でした。導入時は必ずI2Cスキャナー等で実機のアドレスを確認してください。）

```ini

    m5stack/M5GFX @ ^0.2.3
```

`M5Unit-ByteButton`ライブラリは、ByteSwitchユニットの制御にも共用されています。PlatformIO公式レジストリには未登録のため、GitHub直リンクで指定しています。

## 操作方法

### スイッチ（音階）

Byteスイッチユニットの8個のトグルスイッチに、左から低い「ド」〜高い「ド」（1オクターブ）が割り当てられています。

| スイッチ位置 | 音階 | LED色（ON時） |
|---|---|---|
| 一番左 | Do（低） | 赤 |
| 左から2番目 | Re | 橙 |
| 左から3番目 | Mi | 黄 |
| 左から4番目 | Fa | 緑 |
| 左から5番目 | Sol | 青 |
| 左から6番目 | La | 藍 |
| 左から7番目 | Si | 紫 |
| 一番右 | Do（高） | マゼンタ |

スイッチをONにしている間、対応する音が鳴り続けます（トグルスイッチの自然な挙動に合わせています）。複数のスイッチを同時にONにした場合は、より左側（低い音）が優先されます。

### 本体ボタン（音程変更）

M5Stack Basic本体の物理ボタンを押している間、以下のように音程が変化します（複数同時押しも可能です）。

| ボタン | 効果 |
|---|---|
| BtnA（左） | 半音下げる（♭） |
| BtnB（中央） | 半音上げる（♯） |
| BtnC（右） | オクターブ上げる |

## LED表示仕様

ByteSwitchユニットのLEDは手動モード（`BYTE_LED_USER_DEFINED`）で制御しています。各スイッチのON/OFF状態に応じて、対応する音階の色を個別に点灯・消灯させています。

## 既知の注意点・トラブルシューティング履歴

開発時に発生した主な問題と対処法の記録です。同様の問題が発生した場合の参考にしてください。

| 症状 | 原因 | 対処 |
|---|---|---|
| `requestFrom(): i2cWriteReadNonStop returned Error -1` が出続ける | I2Cアドレスの誤り（`0x46`と誤認識） | I2Cスキャナーで実機アドレス（`0x47`）を確認し修正 |
| 起動直後から常に「Do」の音が鳴り続ける | 全スイッチが常時`status=1`を返していた（極性の誤解） | 実機はスイッチ未操作時=1、押下時=0だったため、判定条件を反転 |
| `device.begin() result: 0`、Error 263が出続ける | `device.begin()`のSDAピン引数の誤入力（`21`のつもりが`2`になっていた） | 引数を`21`に修正 |
| ビルド後、`waiting for download`のまま止まる | オートダウンロード回路の自動リセット失敗（既知の症状） | 本体電源ボタンを押す、USB抜き差し、または再アップロードで解消 |
